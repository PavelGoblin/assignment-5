What is the Document Object model (DOM)?

The DOM (Document Object Model) is a programming interface that represents HTML, XML, or other structured documents as a tree of objects. It provides a way for programs to access and manipulate the structure, style, and content of a document.

How do you select on element from the DOM?

Selecting an element from the DOM involves finding the specific element within the document tree based on its unique identifier or properties. There are several common methods to achieve this in JavaScript:

1. By ID:

Use the getElementById() method to select an element based on its unique ID attribute.
const elementById = document.getElementById("myElement");
2. By Class Name:

Use the getElementsByClassName() method to select multiple elements based on their class name. It returns a collection of elements.
const elementsByClassName = document.getElementsByClassName("myClass");
3. By Tag Name:

Use the getElementsByTagName() method to select multiple elements based on their tag name. It returns a collection of elements.
const elementsByTagName = document.getElementsByTagName("p");

4. By Query Selector:

Use the querySelector() method to select a single element based on a CSS selector. It supports a wide range of selectors, including ID, class, tag name, attributes, and more.
const elementByQuerySelector = document.querySelector("#myElement.myClass");

5. By Query Selector All:

Use the querySelectorAll() method to select multiple elements based on a CSS selector. It returns a collection of elements.
const elementsByQuerySelectorAll = document.querySelectorAll(".myClass")

What is event delegation in the context of the DOM, and why is it useful?

Event Delegation in the DOM refers to the practice of attaching an event listener to a parent element instead of directly to each child element. This allows you to efficiently handle events for multiple child elements without having to explicitly add listeners to each one.

Why is it useful?

Improved Performance:
When dealing with a large number of child elements, adding individual event listeners can significantly impact performance. Event delegation reduces the overhead by only attaching a single listener to the parent element.

Dynamic Content:
If the DOM is dynamically updated (e.g., elements are added or removed), event delegation ensures that new elements automatically inherit the event listener from the parent, without requiring manual updates.

Simplified Code:
By using event delegation, you can write more concise and maintainable code, as you only need to manage a single event listener.
